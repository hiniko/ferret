# Full-text search

PostgreSQL full-text (`tsvector` / `tsquery`) with GIN-indexed sidecar columns. Schema is
generated by `Ferret.Migrations` ([migrations.md](migrations.md)).

## Annotate

```csharp
[SearchableEntity]
[SearchGroup("content", FullTextConfig = "english")]
public sealed class Article : ISearchableEntity<Guid>
{
    public Guid Id { get; init; }

    [Searchable(Backend = SearchBackend.FullText, Group = "content", Weight = 2.0f)]
    public string Title { get; init; } = "";

    [Searchable(Backend = SearchBackend.FullText, Group = "content", Weight = 1.0f)]
    public string Body { get; init; } = "";
}
```

## Enable

```csharp
services.AddFerret(o => o
    .ScanAssembly(typeof(Article).Assembly)
    .UseFullTextSearch(ft => ft.DefaultConfig = "english"));
```

## Groups & weights

A property joins a named **group** (e.g. `content`, `tags`); all properties in a group share
one `tsvector` sidecar column. Per-property `Weight` maps to PostgreSQL setweight buckets.
Multiple `[Searchable(..., Group = "...")]` attributes per property are supported — declare
different groups for hybrid coverage ([vector-search.md](vector-search.md),
[hybrid](#hybrid)).

Parser, group selection, and backend choice are configured on `UseFullTextSearch(...)`, not
per-request. The only request-side knob is `PagedQuery.Search`.

## Reindex

The migration backfills per the group's `ReindexMode` (`Inline` / `Concurrent` / `Deferred`)
— see [migrations.md](migrations.md#reindex-modes).

## Hybrid

When an entity declares more than one backend (e.g. full-text + vector), enable
`UseHybridSearch()` to fuse them with weighted Reciprocal Rank Fusion. Per-backend weights and
confidence thresholds via `[HybridBackend(...)]`; force a single backend per query with
`PagedQuery.Backend`.
